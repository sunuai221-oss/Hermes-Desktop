# Security Policy

## Reporting a Vulnerability

Please do not report security issues in public GitHub issues.

If GitHub private vulnerability reporting is available for this repository, use it. Otherwise, contact the maintainer directly through GitHub and provide the details privately.

Please include:

- a short description of the issue
- affected files or commands
- reproduction steps
- expected impact
- any suggested mitigation, if you have one

## Scope

Relevant issues include:

- command execution or path-handling problems
- unsafe file access
- authentication or authorization problems
- secrets exposure
- dependency issues that can affect the packaged desktop app or local backend

## Response Approach

The goal is to confirm, contain, fix, and disclose responsibly.

Please allow time for triage and remediation before any public disclosure.
